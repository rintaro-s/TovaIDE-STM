# TovaIDE 深層統合設計

## 現状の問題
- 単なる VS Code 拡張機能の集合
- VS Code の UI/UX をそのまま使用
- STM32 開発に特化した統合感がない

## 目標
VS Code のコア機能を活用しつつ、STM32 開発に最適化された専用 IDE を構築

---

## Phase 1: カスタムワークベンチ統合 (即座に実装可能)

### 1.1 カスタムウェルカム画面
**現状**: VS Code のデフォルトウェルカム画面
**改善**: STM32 専用ダッシュボード

```typescript
// src/vs/workbench/contrib/welcome/page/browser/welcomePage.ts を拡張
// TovaIDE 専用のウェルカム画面を実装
```

**内容**:
- 最近のプロジェクト (`.ioc` ファイル自動検出)
- クイックアクション (新規プロジェクト / テンプレート / ビルド / 書き込み)
- ツールステータス (CubeMX / CubeCLT / Programmer / GCC)
- ハードウェア接続状態 (ST-LINK リアルタイム検出)

### 1.2 カスタムタイトルバー
**現状**: 標準 VS Code タイトルバー
**改善**: STM32 専用情報表示

```typescript
// src/vs/workbench/browser/parts/titlebar/titlebarPart.ts を拡張
```

**追加要素**:
- MCU セレクタ (現在の `.ioc` から自動検出)
- ビルド構成 (Debug / Release)
- ST-LINK 接続インジケーター (緑/赤)
- クイックビルド/書き込みボタン

### 1.3 カスタムステータスバー
**現状**: 汎用ステータスバー
**改善**: STM32 開発情報に特化

**左側**:
- プロジェクト名 + MCU 型番
- ビルド状態 (ビルド中 / 成功 / エラー数)
- フラッシュ使用率 (例: `Flash: 45.2KB / 512KB (8.8%)`)
- RAM 使用率 (例: `RAM: 12.1KB / 128KB (9.5%)`)

**右側**:
- ST-LINK 状態 (接続中 / 未接続)
- デバッグ状態 (実行中 / 停止中 / ブレークポイント)
- コラボレーション状態 (同期中のクライアント数)

## Phase 3: カスタムビュー統合

### 3.1 専用サイドバー
**現状**: 汎用アクティビティバー
**改善**: STM32 専用サイドバー

**構成**:
```
┌─────────────────────┐
│ ⬡ プロジェクト      │ ← ファイルツリー + .ioc 優先表示
│ ⚙ ピン設定          │ ← ピンビジュアライザ統合
│ 🔨 ビルド           │ ← ビルド設定 + ログ
│ 🐛 デバッグ         │ ← レジスタ + メモリ + SWV
│ 🤖 AI アシスタント  │ ← チャット + 自動化
│ 🌐 コラボレーション │ ← リアルタイム同期
└─────────────────────┘
```

### 3.2 統合デバッグビュー
**現状**: 汎用デバッグビュー
**改善**: STM32 専用デバッグビュー

**機能**:
- SVD レジスタビュー (ビット単位編集)
- メモリビュー (16進/ASCII 表示)
- SWV トレースビュー (リアルタイム)
- ペリフェラル状態ビュー (UART/SPI/I2C/ADC)

```
│                                         │

└─────────────────────────────────────────┘
```

### 5.2 自動ツール検出とセットアップ
**初回起動時**:
1. CubeMX / CubeCLT を自動検出
2. 未検出の場合、ダウンロードリンクを表示
3. インストール後、自動的にパスを設定
4. PATH に GCC / Programmer を自動追加

## 技術的アプローチ

### VS Code コア修正箇所
```
src/vs/workbench/
├─ browser/
│  ├─ parts/titlebar/     ← タイトルバー
│  ├─ parts/statusbar/    ← ステータスバー
│  └─ parts/sidebar/      ← サイドバー
├─ contrib/
│  ├─ welcome/            ← ウェルカム画面
│  ├─ debug/              ← デバッグビュー
│  └─ markers/            ← エラー表示
└─ services/
   ├─ editor/             ← エディター統合
   └─ keybinding/         ← キーバインド
```

### 拡張機能との連携
- コア UI は VS Code 本体を修正
- ビジネスロジックは拡張機能に実装
- `vscode.commands` で連携
