## 1. プロジェクトビジョン

### 1.1 コンセプト

**「STM32開発を、世界で最も気持ちよくする。」**

CubeForge IDEは VSCode (MIT: code-oss) をベースにSTM32専用にフォークした次世代組込み開発環境。Eclipse由来のSTM32CubeIDEが持つ**全機能を完全網羅**しつつ、以下の3軸で圧倒的な差別化を実現する。

| 差別化軸 | 内容 |
|---|---|
| **AIネイティブ** | 要件を日本語で話しかけるだけで、ピン設定→コード生成→書込みまで全自動 |
| **モダンUX** | Eclipse感・ゲーミング感ゼロ。Figma / Linear / Notionレベルのデザイン |
| **日本語ファースト** | UI・エラー・ドキュメント・AIの応答がすべて日本語ネイティブ |

### 1.2 技術スタック

```
┌─────────────────────────────────────────────────────┐
│  CubeForge IDE (Electron + Node.js)                  │
│  ┌─────────────────┐  ┌──────────────────────────┐  │
│  │  VSCode core    │  │  React Webview Panels    │  │
│  │  (code-oss MIT) │  │  (Tailwind + shadcn/ui)  │  │
│  └────────┬────────┘  └──────────────────────────┘  │
│           │                                           │
│  ┌────────▼──────────────────────────────────────┐  │
│  │            Extension Host (Node.js)            │  │
│  │  ┌──────────┐ ┌──────────┐ ┌───────────────┐  │  │
│  │  │stm32-core│ │stm32-ai  │ │stm32-collab   │  │  │
│  │  └──────────┘ └──────────┘ └───────────────┘  │  │
│  └────────────────────────┬──────────────────────┘  │
└───────────────────────────┼─────────────────────────┘
                            │ subprocess / IPC
        ┌───────────────────┼───────────────────┐
        ▼                   ▼                   ▼
  STM32CubeMX         STM32CubeCLT        MCP Server
  (GUI + CLI)    (make / GDB / CLIProg)  (localhost:3737)
```

**言語・フレームワーク:**
- メイン: TypeScript 5.x (Extension), React 18 (Webview)
- スタイリング: Tailwind CSS 3 (Webview), CSS Custom Properties (IDE本体)
- ビルドツール: esbuild (Extension), Vite (Webview)
- テスト: Vitest + Playwright (E2E)

## 5. エディタエリア詳細

### 5.1 タブバー

```
┌──────────────────────────────────────────────────────────────────────┐
│ [main.c ×] [gpio.h] [MyProject.ioc ●] [CMakeLists.txt]  [⊕]  [⊟⊠]  │
└──────────────────────────────────────────────────────────────────────┘
```

- 変更あり: タブ名右に `●` (オレンジ)
- iocファイル: タブアイコンにSTM32ロゴ
- デバッグ中の実行行タブ: タブ背景を薄い緑
- タブ右クリックメニュー: 閉じる/他を閉じる/右を閉じる/パスをコピー/分割

### 5.2 C/C++ エディタ拡張機能

#### ガター (行番号左側)

```
┌────────────────────────────────────────
│ [変更差分] [ブレークポイント] [行番号]  コード本文
│   ▏               ○         23 │  HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_SET);
│   ▏               ●         24 │  HAL_Delay(500);
│   ┆                         25 │  HAL_GPIO_WritePin(GPIOB, GPIO_PIN_0, GPIO_PIN_RESET);
└────────────────────────────────────────
  ○ = ブレークポイント未設定 (ホバーで表示)
  ● = ブレークポイント設定済み (赤丸)
  ◆ = 条件付きブレークポイント (黄ダイヤ)
  ▏ = Git変更 (緑=追加, 青=変更, 赤=削除)
```

#### インラインデコレーション (STM32専用)

```c
// 16進数リテラルプレビュー
uint32_t mask = 0xFF00FF00;          // → 右に薄く「= 4278255360」
uint8_t  pins = 0b11001100;          // → 右に薄く「= 204」

// ビット演算プレビュー
GPIOB->ODR |= (1 << 5);             // → 右に薄く「bit5: PA5 → 設定済み」

// HAL関数ドキュメントホバー
HAL_UART_Transmit(...)               // → ホバーで引数・戻り値・説明をポップアップ

// User Code セクション強調
/* USER CODE BEGIN 2 */              // → 背景色 accent/5 でハイライト (再生成安全域)
  my_init();
/* USER CODE END 2 */

// ペリフェラルレジスタ定数
USART_CR1_UE                        // → ホバーで SVD由来の説明「USART Enable bit」
```

#### コード補完 (Clangd拡張)

- HAL関数: 第1引数にハンドル型を自動推論
- パラメータヒント: 関数入力中に引数名+型をグレーで表示
- スニペット展開: `hal_uart_tx` + Tab → UART送信コードブロック展開
- iocファイルとの整合性チェック: 未有効化のペリフェラルを使用すると波線 + 「CubeMXでUART2を有効化してください」

#### 分割ビュー

```
┌─────────────────────────┬─────────────────────────┐
│  main.c (コード)         │  MyProject.ioc (ピン設定)│
│                          │  [ピン設定ビジュアライザ] │
└─────────────────────────┴─────────────────────────┘
```

- `Ctrl+\` で左右分割
- `Ctrl+Shift+\` で上下分割
- iocファイルを開くと右ペインに自動でピン設定ビジュアライザを提案

## 17. STM32CubeMX統合

### 17.1 CubeMX GUI埋め込み

**実装方式:** Electron BrowserView を使ったネイティブウィンドウ埋め込み

```
手順:
1. エディタエリアで .ioc ファイルを開く
2. CubeForge が STM32CubeMX プロセスを起動 (または既存を利用)
3. CubeMX の ウィンドウ を BrowserView/WebContentsView として
   エディタペインに埋め込み表示
4. CubeMX の保存イベントを監視 (iocファイルのmtime polling or FSWatcher)
5. 変更検知 → 「コードを再生成しますか？」通知バナー

代替手段 (埋め込み困難な場合):
  - CubeMXを別ウィンドウで起動
  - iocファイルの変更を監視して再生成を提案
```

### 17.2 CubeMX CLIバッチスクリプト

**スクリプトテンプレート (mxscript形式):**

### 19.3 MCPツール呼び出しセキュリティ

```
1. プロジェクトルート外のファイルアクセス禁止
   → パスを resolve() して projectRoot 配下か検証

2. 破壊的操作 (erase_all, flash) には confirmed: true 必須
   → AIエージェントが誤って全消去しないガード

3. レート制限: flash は 1回/10秒、build は 1回/5秒

4. 全ツール呼び出しを MCPログタブ (§12.6) に記録
   → AIが何をしたか完全に追跡可能

5. MCPトークンはOS keychain保存、平文禁止
```

## 21. AIエンドtoエンド自動化フロー

### 21.1 「半自動」モードのフロー (推奨デフォルト)

```
ユーザー入力: 「UART2で"Hello World"を送信するコードを書いて」
  │
  ├─ AI: コンテキスト収集
  │   └─ ioc確認 → UART2有効✅, PA2:TX, PA3:RX
  │
  ├─ AI: コード生成
  │   └─ main.c の USER CODE BEGIN 3 に挿入するコードを生成
  │
  ├─ UI: 変更プレビュー表示 (§10.1)
  │   └─ ユーザーが [適用する] をクリック
  │
  ├─ ファイル書込み (User code セクション内のみ)
  │
  ├─ AI: ビルド実行 (MCP: build)
  │   ├─ 成功 → 次へ
  │   └─ 失敗 → AIが自動修正を試みる (最大3回)
  │
  ├─ UI: 書込み確認ダイアログ  ← ここで必ずユーザー確認
  │   「ビルド成功しました。書込みますか？」[書込む] [後で]
  │
  └─ [書込む] → MCP: flash() → 完了通知
```

### 21.2 「全自動」モードのフロー

> **⚠️ 設定で明示的に有効化が必要。デフォルトOFF**

```
ユーザー入力: 「コンパイルして書込んで」
  │
  ├─ AI: build() → 成功
  ├─ AI: flash() → 完了
  └─ 通知: 「✅ ビルド+書込み完了 (12.3 + 2.1 = 14.4秒)」
```

### 21.3 ハードフォルト自動解析フロー

```
デバッグ中に HardFault 発生
  │
  ├─ Fault Analyzer 自動起動 (§9.5)
  ├─ CFSR/HFSR/MMFAR/BFAR を自動読取
  │
  ├─ AI: レジスタ値 + コールスタック + 直前の変数値を収集
  │
  ├─ MCP: get_fault_analysis() → AIに送信
  │
  └─ AIパネルに日本語解析結果表示:
     「UsageFaultです。スタックオーバーフローの可能性があります。

      [FreeRTOSタスク設定を開く]」

## 28. 非機能要件

### 28.1 パフォーマンス

| 指標 | 目標値 | 測定方法 |

| コールドスタート起動時間 | 5秒以内 (SSD) | 空プロジェクト起動 |
| アイドル時メモリ | 400MB以下 | Task Manager / Activity Monitor |
| ビルドオーバーヘッド | 200ms以内 | make実行前後の差分 |
| コード補完応答 | 200ms以内 | Clangd応答時間 |
| UI フレームレート | 60fps | Chrome DevTools |
| 書込み速度 | CubeProgrammer-CLI と同等 | 50KBのELFで測定 |

### 28.2 セキュリティ

```
1. MCPサーバーは localhost:3737 のみバインド
   → 0.0.0.0バインド禁止。外部アクセス不可

2. MCPトークンはOS keychain保存
   → Windows: DPAPI / macOS: Keychain / Linux: libsecret
   → 平文ファイル保存禁止

3. LAN共有は mDNSスコープ (リンクローカル) のみ
   → WAN公開機能なし

4. GitデーモンはLAN内のみ
   → 認証なしのため外部公開禁止

5. AIへのコンテキスト送信はユーザー設定で制御
   → 機密コードの外部送信を制御可能

6. ファイルアクセス制限
   → MCPツールはプロジェクトルート以外へのアクセス禁止
```

### 28.3 アクセシビリティ

```
- 全インタラクティブ要素に aria-label 必須
- キーボードのみで全機能が操作可能
- フォーカスリング: 2px solid var(--cf-border-focus)

- カラーコントラスト比: WCAG AA (4.5:1) 準拠
- スクリーンリーダー: NVDA / VoiceOver 動作確認

## 付録: 用語集

| 用語 | 説明 |
|---|---|
| CubeCLT | STM32CubeCLT — ST社のコマンドラインツールセット。GCC/GDB/書込みツールを内包 |
| CubeMX | STM32CubeMX — GUIでSTM32のピン/クロック/ミドルウェアを設定しCコードを生成するツール |
| ioc | STM32CubeMXのプロジェクトファイル。ピン設定・クロック設定が記述されている |
| MCP | Model Context Protocol — AIエージェントがツールを呼び出すためのプロトコル |
| DAP | Debug Adapter Protocol — VSCodeのデバッガプロトコル |
| SWD | Serial Wire Debug — STM32のデバッグ/書込みインターフェース (2ピン) |
| SWO | Serial Wire Output — SWDに追加する1ピンのトレース出力 |
| SWV | Serial Wire Viewer — SWD+SWOを使ったリアルタイムトレース機能 |
| ITM | Instrumentation Trace Macrocell — printf等のデータを出力するCortex-M機能 |
| SVD | System View Description — MCUのペリフェラルレジスタ定義XML |
| CRDT | Conflict-free Replicated Data Type — リアルタイム共同編集に使うデータ構造 |
| User code セクション | `/* USER CODE BEGIN */` 〜 `/* USER CODE END */` で囲まれた、CubeMX再生成で保護されるコード領域 |
| HAL | Hardware Abstraction Layer — ST社が提供するSTM32向けCライブラリ群 |
| LL | Low-Layer — HALより低レベルなSTM32ライブラリ (高速・高効率) |
